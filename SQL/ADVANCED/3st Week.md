# 1️⃣ 학습 내용 정리

## 1. MySQL의 데이터 형식

### 1-1. 정수형
소수점이 없는 숫자(인원 수, 가격, 수량 등)를 저장할 때 사용합니다.
* **종류와 크기 :** 크기가 작은 순서대로 `TINYINT`(1바이트), `SMALLINT`(2바이트), `INT`(4바이트), `BIGINT`(8바이트)가 있습니다.
* **`UNSIGNED` 예약어 :** 값의 범위가 0부터 시작하도록 변경해 주는 키워드입니다. 예를 들어 `TINYINT`는 기본적으로 -128 ~ 127까지 저장하지만, `TINYINT UNSIGNED`로 지정하면 음수 범위가 사라지는 대신 0 ~ 255까지 저장할 수 있어 양수 표현 범위를 2배로 늘릴 수 있습니다.

### 1-2. 문자형
글자를 저장하기 위해 사용하며, 고정길이와 가변길이로 나뉩니다.
* **`CHAR` (고정길이 문자형) :** 자릿수가 고정되어 있습니다. 예를 들어 `CHAR(10)`에 '가나다' 3글자만 저장해도 무조건 10자리의 공간을 모두 차지하므로 공간이 낭비될 수 있지만, MySQL 내부적인 처리 성능(속도)은 더 좋습니다.
* **`VARCHAR` (가변길이 문자형) :** 글자의 길이에 맞춰 공간이 변합니다. `VARCHAR(10)`에 '가나다'를 넣으면 딱 3자리 분량의 공간만 사용하므로 공간 효율성이 뛰어납니다.

### 1-3. 대량의 데이터 형식
문자형인 `CHAR`(최대 255자)나 `VARCHAR`(최대 16383자)로 감당할 수 없는 아주 큰 데이터를 저장할 때 사용합니다.
* **텍스트 데이터 :** 소설이나 영화 대본 같이 길이가 엄청나게 긴 글자를 저장할 때는 `TEXT` 또는 `LONGTEXT`를 사용합니다.
* **바이트/이진 데이터 :** 이미지, 동영상 등의 데이터는 글자가 아닌 이진(Binary) 데이터이므로, 이를 저장할 때는 `BLOB` 또는 `LONGBLOB`을 사용해야 합니다.

### 1-4. 실수형 및 날짜형
* **실수형 :** 소수점이 있는 숫자를 저장하며 `FLOAT`(소수점 아래 7자리), `DOUBLE`(소수점 아래 15자리)이 있습니다.
* **날짜형**
  * `DATE` : 날짜만 저장 (YYYY-MM-DD 형식).
  * `TIME` : 시간만 저장 (HH:MM:SS 형식).
  * `DATETIME` : 날짜와 시간을 모두 저장 (YYYY-MM-DD HH:MM:SS 형식).

### 1-5. 변수의 사용
SQL에서도 다른 프로그래밍 언어처럼 일시적으로 변수를 선언하고 값을 대입해 사용할 수 있습니다.
* **선언 및 대입 :** `SET @변수이름 = 변수의 값 ;`.
* **출력 :** `SELECT @변수이름 ;`.

### 1-6. 데이터 형 변환
문자형을 정수형으로 바꾸거나, 반대로 정수형을 문자형으로 바꾸는 등 데이터의 형식을 변환하는 기능입니다.
* **명시적인 변환 :** 사용자가 직접 함수를 이용해 변환하는 것으로, `CAST()` 또는 `CONVERT()` 함수를 사용합니다. 두 함수는 형식만 다를 뿐 동일한 기능을 합니다.
* **암시적인 변환 :** 함수를 사용하지 않아도 MySQL이 알아서 자연스럽게 형을 변환해 주는 것을 의미합니다. (예: 문자로 된 `'100'`과 `'200'`을 더하는 연산을 하면, 자동으로 숫자 100과 200으로 변환해 덧셈을 수행합니다.)

## 2. 두 테이블을 묶는 조인

### 2-1. 조인의 필수 조건 : 일대다(1:N) 관계
두 테이블을 조인하기 위해서는 테이블들이 **일대다(one to many) 관계**로 연결되어 있어야 합니다.
* **개념 :** 한쪽 테이블에는 데이터가 하나만 존재해야 하고, 연결된 다른 테이블에는 여러 개의 데이터가 존재할 수 있는 관계입니다 

    (EX. 회원 테이블의 회원은 1명이지만, 구매 테이블에서 해당 회원의 구매 내역은 여러 개일 수 있음).
* **기본 키(PK)와 외래 키(FK) :** 주로 1쪽 테이블의 기본 키(Primary Key)와 다(N)쪽 테이블의 외래 키(Foreign Key)가 맺어져서 일대다 관계를 형성합니다.

### 2-2. 내부 조인 (Inner Join)
가장 많이 사용되는 기본적인 조인 방식입니다. 그냥 '조인'이라고 부르면 보통 내부 조인을 의미합니다.
* **특징 :** 두 테이블에 **모두 데이터가 존재하는 경우에만** 결과를 추출합니다. 

    (EX. 회원 가입을 한 사람 중에서 물건을 구매한 이력이 있는 회원만 조회)
* **기본 형식** 
  ```sql
  SELECT <열 목록>
  FROM <첫 번째 테이블>
      INNER JOIN <두 번째 테이블>
      ON <조인될 조건>
  [WHERE 검색 조건] ;
  ```
  *(참고: `INNER JOIN`을 그냥 `JOIN`이라고만 써도 MySQL이 알아서 내부 조인으로 인식합니다.)*

### 2-3. 외부 조인 (Outer Join)
내부 조인과 달리, 양쪽에 데이터가 다 있지 않고 **한쪽 테이블에만 데이터가 있어도 결과를 추출**할 수 있는 방식입니다. 
* **`LEFT OUTER JOIN` :** '왼쪽 테이블'의 모든 내용은 빠짐없이 출력되어야 한다는 의미입니다.
* **`RIGHT OUTER JOIN` :** '오른쪽 테이블'의 모든 내용은 빠짐없이 출력되어야 한다는 의미입니다.
* **`FULL OUTER JOIN` :** 왼쪽과 오른쪽 외부 조인이 합쳐진 것으로, 양쪽 테이블의 모든 데이터가 빠짐없이 출력됩니다.

### 2-4. 기타 조인 (상호 조인 & 자체 조인)
자주 사용되지는 않지만 알아두어야 할 특수한 조인 방식들입니다.

* **상호 조인 (Cross Join)** 
  * 한쪽 테이블의 모든 행과 다른 쪽 테이블의 모든 행을 무조건 하나씩 다 엮는 방식입니다 (카티션 곱이라고도 부름).
  * 결과로 나오는 행의 개수는 `두 테이블의 행의 개수를 곱한 값`이 됩니다.
  * 실제 의미 있는 데이터를 뽑기 위해서라기보다는, 시스템 성능 테스트를 위해 **대용량의 가짜 데이터를 순간적으로 만들어낼 때** 주로 사용합니다.
* **자체 조인 (Self Join)** 
  * 2개의 서로 다른 테이블을 엮는 것이 아니라, **자기 자신(1개의 테이블)과 조인**하는 방식입니다.
  * 하나의 테이블 안에 같은 성격의 열이 존재할 때 사용하며, 대표적으로 회사 직원의 '직속 상관'을 찾을 때와 같은 **조직도 구조**를 표현할 때 사용됩니다.

| 용어 | 설명 |
| :--- | :--- |


```
2. LIMIT B.prod_name IS NULL
```

## 3. SQL 프로그래밍 

### 3-1. 조건에 따른 분기 : `IF` 문
`IF` 문은 가장 기본적으로 사용되는 조건 제어문으로, 조건식이 참(True)인지 거짓(False)인지에 따라 실행할 SQL 문장을 결정합니다.

* **기본 형식:**
  ```sql
  IF <조건식> THEN
      SQL문장들
  END IF;
  ```
* **`IF ~ ELSE` 문 :** 조건식이 참일 때와 거짓일 때 실행할 문장을 나눌 수 있습니다.
* **특징 :** 두 문장 이상이 처리되어야 할 때는 반드시 `BEGIN ~ END`로 묶어주어야 합니다. 습관적으로 `BEGIN ~ END`를 묶어주는 것을 권장합니다.
* **변수 활용 :** `DECLARE`로 변수를 선언하고, `SET`으로 값을 대입하여 `IF` 문의 조건식에 활용할 수 있습니다.

### 3-2. 다중 분기 : `CASE` 문
`IF` 문은 참 아니면 거짓 두 가지만 있기 때문에 2중 분기라고 부르지만, 조건이 여러 개일 경우 `CASE` 문을 사용하여 **'다중 분기'**를 구현합니다.

* **기본 형식:**
  ```sql
  CASE
      WHEN 조건1 THEN SQL문장들1
      WHEN 조건2 THEN SQL문장들2
      ELSE SQL문장들3
  END CASE;
  ```
* **특징 :** 여러 조건에 해당하지 않으면 제일 마지막의 `ELSE` 부분을 수행합니다.
* **활용 :** 90점 이상은 A, 80점 이상은 B를 부여하는 학점 계산이나, 총 구매액에 따라 '최우수고객', '우수고객', '유령고객' 등 회원 등급을 나누는 작업에 매우 유용하게 쓰입니다.

### 3-3. 반복 처리 : `WHILE` 문
조건식이 참인 동안 특정한 SQL 문장들을 계속 반복해서 실행해야 할 때 사용합니다.

* **기본 형식:**
  ```sql
  WHILE <조건식> DO
      SQL 문장들
  END WHILE;
  ```
* **반복문 제어 키워드 (응용) :** 1부터 100까지 더하다가 합계가 특정 숫자를 넘으면 반복문을 빠져나가는 등의 처리를 위해 다음 키워드들을 사용합니다.
  * **`ITERATE [레이블]` :** 지정한 레이블로 가서 반복문을 계속 진행합니다 (일반 프로그래밍 언어의 `CONTINUE` 역할).
  * **`LEAVE [레이블]` :** 지정한 레이블의 반복문을 빠져나갑니다 (일반 프로그래밍 언어의 `BREAK` 역할).

### 3-4. 실행 시점에 결정되는 : `동적 SQL`
SQL 문을 미리 고정해두는 것이 아니라, 상황에 따라 내용이 변경될 때 동적으로 SQL을 생성하여 실행하는 기능입니다.

* **동적 SQL의 3단계 흐름**
  1. **`PREPARE` :** SQL 문을 실행하지 않고 미리 준비만 해놓습니다. 값이 들어갈 자리는 물음표(`?`)로 비워둘 수 있습니다.
     *(예: `PREPARE myQuery FROM 'SELECT * FROM member WHERE mem_id = ?';`)*
  2. **`EXECUTE` :** 준비된 SQL 문을 실행합니다. `?` 자리에 들어갈 값은 `USING` 키워드를 통해 변수로 전달할 수 있습니다.
     *(예: `EXECUTE myQuery USING @변수이름;`)*
  3. **`DEALLOCATE PREPARE` :** 실행이 끝난 후, 준비했던 문장을 메모리에서 해제합니다.
* **활용 :** 보안이 중요한 출입문에서 출입한 내역을 테이블에 기록할 때, 실행 시점의 현재 날짜와 시간을 동적으로 입력하는 등의 작업에 유용하게 쓰입니다.

### 💡 [요약] SQL 프로그래밍 키워드
| 키워드 | 설명 |
| :--- | :--- |
| **IF / ELSE** | 조건식의 참/거짓에 따라 두 갈래로 분기할 때 사용 |
| **CASE** | 여러 가지 조건 중 하나를 선택하는 다중 분기에 사용 |
| **WHILE** | 조건식이 참인 동안 SQL 문을 반복 실행 |
| **ITERATE / LEAVE** | WHILE 문 내부에서 반복을 건너뛰거나 종료할 때 사용 |
| **PREPARE / EXECUTE** | 동적 SQL을 미리 준비하고 실행할 때 사용 (변수는 USING으로 전달) |

> **확인문제 : 다음은 CASE 문의 형식입니다. 빈칸에 들어갈 가장 적절한 명령어를 보기에서 고르세요..**

```sql
CASE
    (1) 조건 THEN
        SQL문장들1
    ELSE
        SQL문장들4
END (2);
```

보기는 아래와 같습니다.
```
WHEN / THEN / CURRENT / DATE / TIME / IF / END IF / CASE
```

```
(1) WHEN
(2) CASE
```
