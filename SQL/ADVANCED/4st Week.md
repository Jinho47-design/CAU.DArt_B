## 2. 제약조건으로 테이블을 견고하게 

> 제약조건(Constraint) 핵심 정리

### 2-1. 제약조건의 개념과 목적
* **데이터의 무결성 :** 데이터에 '결함이 없는 상태'를 뜻합니다. 
* **역할 :** 중복된 이메일이나 가입되지 않은 아이디로 구매하는 등 데이터의 결함이 발생하는 것을 방지하여 데이터의 무결성을 지키는 규칙

### 2-2. 6가지 핵심 제약조건

### ① 기본 키 제약조건 (PRIMARY KEY)
* **개념 :** 테이블의 많은 행 데이터 중에서 데이터를 구분할 수 있는 고유한 식별자 역할을 합니다 (예: 학번, 사번, 주민등록번호 등).
* **특징** 
  * 입력되는 값은 절대 **중복될 수 없으며, 비어있어도(NULL) 안 됩니다**.
  * 테이블당 **단 1개**만 설정할 수 있습니다.
  * 기본 키로 지정된 열은 자동으로 클러스터형 인덱스(찾아보기)가 생성됩니다.

### ② 외래 키 제약조건 (FOREIGN KEY)
* **개념 :** 두 테이블 사이의 관계를 연결해주고 데이터의 무결성을 보장해주는 역할입니다.
* **구조 :** 보통 회원 테이블의 기본 키(기준 테이블)와 구매 테이블의 외래 키(참조 테이블)가 연결되어 일대다 관계를 맺습니다.
* **CASCADE 옵션 :** 기준 테이블의 데이터가 변경/삭제될 때 참조 테이블의 데이터도 함께 자동으로 변경/삭제되도록 해주는 `ON UPDATE CASCADE`와 `ON DELETE CASCADE` 구문을 유용하게 사용할 수 있습니다.

### ③ 고유 키 제약조건 (UNIQUE)
* **개념 :** 기본 키와 거의 비슷하게 '중복되지 않는 유일한 값'을 입력해야 하는 조건입니다.
* **차이점 :** 기본 키와 달리 **NULL(빈 값)을 허용**하며, 하나의 테이블에 **여러 개를 설정**할 수 있습니다. (예: 회원 테이블에서 기본 키는 아이디로 하되, 이메일 주소도 중복되지 않아야 하므로 이메일 열에 고유 키를 설정).

### ④ 체크 제약조건 (CHECK)
* **개념 :** 테이블에 입력되는 데이터를 사전에 점검하는 기능입니다.
* **활용 :** 키(height)에 마이너스 값이 들어오지 않도록 '100 이상만 가능'하게 하거나, 전화번호 국번을 '02', '031' 등 특정 값만 들어오도록 값의 범위를 제한할 때 사용합니다,. 

### ⑤ 기본값 정의 (DEFAULT)
* **개념:** 데이터를 입력할 때 실수로 값을 생략하더라도, 자동으로 입력될 값을 미리 지정해 놓는 방법입니다.
* **활용:** 연락처 국번을 입력하지 않으면 기본으로 '02'가 들어가도록 설정할 수 있습니다.

### ⑥ 널 값 허용 (NULL / NOT NULL)
* **개념:** 데이터에 빈 값을 허용할지 말지 결정합니다. `NULL`은 '아무것도 없다'는 뜻이며 공백이나 0과는 다릅니다.
* **특징:** `NOT NULL`을 설정하면 반드시 값을 입력해야 합니다. `PRIMARY KEY`가 설정된 열은 자동으로 `NOT NULL`로 인식됩니다.

### 💡 [요약] 한눈에 보는 제약조건 키워드
| 제약조건 종류 | 예약어 | 핵심 설명 |
| :--- | :--- | :--- |
| **기본 키** | `PRIMARY KEY` | 행을 구분하는 식별자 (중복 불가, NULL 불가, 테이블당 1개) |
| **외래 키** | `FOREIGN KEY` | 두 테이블을 연결하여 무결성을 보장하는 키 |
| **고유 키** | `UNIQUE` | 중복되지 않는 값 (NULL 허용, 여러 개 설정 가능) |
| **체크** | `CHECK` | 조건에 맞는 데이터만 입력되도록 점검하는 기능 |
| **기본값** | `DEFAULT` | 값을 입력하지 않았을 때 자동으로 들어갈 값 지정 |
| **널 허용 여부** | `NULL` / `NOT NULL` | 빈 값의 허용 여부 결정 |

## 2. 실습문제

1. 다음 조건을 만족하는 `users` 테이블을 생성하시오.
```
- user_id는 INT이며 **기본키(Primary Key)**로 설정합니다.
- name은 VARCHAR(20)이며 NULL을 허용하지 않습니다.
- email은 VARCHAR(50)이며 중복을 허용하지 않습니다.
- signup_date는 DATE 타입으로 설정합니다.
- grade는 INT이며 기본값(Default)을 1로 설정합니다.
```

2. 다음 조건을 만족하는 orders 테이블을 생성하시오.
```
- order_id는 INT이며 기본키(Primary Key)로 설정합니다.
- user_id는 INT이며 NULL을 허용하지 않습니다.
- amount는 INT이며 0보다 커야 합니다.
- order_date는 DATE 타입으로 설정합니다.
```

3. 다음 조건을 만족하여 데이터를 삽입하시오.
```
- users 테이블에 3명 이상의 데이터를 직접 INSERT 하시오.
- orders 테이블에 3건 이상의 데이터를 직접 INSERT 하시오.
```

4. users와 orders 테이블을 활용하여 다음 컬럼을 보여주는 뷰 user_order_view를 생성하시오.
```
- user_id
- name
- amount
```

5. 생성한 user_order_view를 조회하시오.
