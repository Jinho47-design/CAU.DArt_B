# 1️⃣ 학습 내용 정리

## 1. 데이터베이스 알아보기

<!-- 데이터베이스와 DBMS에 관해 배우게 된 점을 적어주세요. -->


## 1. 데이터베이스 (Database, DB)
* **정의 :** '데이터의 집합(저장소)'을 의미
* **특징 :** 우리의 일상생활 대부분의 정보(카카오톡 메시지, 인스타그램 사진, 교통카드 승하차 기록, 카페 결제 내역 등)가 모두 데이터베이스에 저장되고 관리됨.

## 2. DBMS (Database Management System)
* **정의 :** 데이터베이스를 **관리하고 운영하는 소프트웨어**입니다.
* **핵심 역할 :** 대용량의 데이터를 관리하며, **여러 명의 사용자나 응용 프로그램이 데이터를 공유하고 동시에 접근**할 수 있도록 해줍니다.
* **DBMS의 필요성 :** 과거에는 종이나 컴퓨터 파일(엑셀 등)로 데이터를 관리했지만, 여러 사람이 동시에 작업하거나 대량의 데이터를 처리할 때 속도가 느리고 오류가 발생하는 한계가 있었습니다. DBMS는 이러한 파일의 단점을 보완하여 데이터를 안정적이고 효율적으로 운영하기 위해 등장했습니다.

## 3. DBMS의 분류와 RDBMS
DBMS는 계층형, 망형, 관계형 등으로 분류되며, 현재 가장 많이 사용되는 형태는 관계형 DBMS(RDBMS)입니다.
* **RDBMS (Relational DBMS) :** 대부분의 DBMS가 채택하고 있는 형태로, 모든 데이터가 테이블(Table)에 저장됩니다.

    * **테이블(Table) :** 데이터베이스의 최소 단위로, 하나 이상의 열(Column)과 행(Row)으로 이루어진 2차원 구조를 갖습니다.

## 4. 데이터베이스를 조작하는 언어 : SQL
* **정의 :** SQL(Structured Query Language)은 관계형 데이터베이스에서 데이터를 구축, 관리, 활용하기 위해 사용되는 **구조화된 질의 언어**입니다.
* **역할 :** 사용자가 DBMS를 통해 중요한 정보를 입력, 관리, 추출할 수 있도록 DBMS와 소통하는 수단이 됩니다. 
* **표준 SQL :** 국제표준화기구에서 정한 표준 SQL이 있으며, 이를 익히면 다양한 회사의 DBMS를 공통적으로 다룰 수 있습니다.

## 5. 대표적인 DBMS 종류
특정 목적을 위해 엑셀이나 포토샵을 설치하듯, 데이터베이스를 사용하기 위해서도 DBMS 소프트웨어를 설치해야 합니다.
* **대표적인 제품 :** MySQL, Oracle, SQL Server, MariaDB, PostgreSQL, SQLite 등이 있습니다.
* **MySQL :** 오픈 소스(무료)로 제공되며, 비교적 사용이 쉬우면서도 실무에서 인기가 많은 대표적인 관계형 DBMS입니다.

## 3. 건물을 짓기 위한 설계도: 데이터베이스 모델링

<!-- 데이터베이스 모델링에 관해 배우게 된 점을 적어주세요. -->

## 1. 데이터베이스 모델링의 개념
* **정의 :** 우리가 살고 있는 현실 세계에서 사용되는 사물이나 작업을 DBMS의 데이터베이스 개체로 옮기기 위한 과정입니다. 
* **핵심 역할 :** 더 쉽게 말해, 현실에서 쓰이는 다양한 정보들을 데이터베이스 내의 **테이블(표) 구조로 변경하기 위해 미리 설계하는 작업**입니다. 
* **비유 :** 건물을 지을 때 설계도가 아주 중요하듯, 데이터베이스를 제대로 구축하기 위해 '건축 설계도를 그리는 과정'과 비슷합니다.
* **특징 :** 건물의 설계도처럼 사람마다 다양한 결과물이 나올 수 있으므로, 데이터베이스 모델링에는 '정답이 없다'는 특징이 있습니다.

## 2. 프로젝트와 폭포수 모델 
데이터베이스 모델링을 이해하려면 먼저 소프트웨어를 개발하는 '프로젝트'의 진행 단계를 알아야 합니다.
* **프로젝트(Project) :** 현실 세계에서 일어나는 업무를 컴퓨터 시스템으로 옮겨놓는 일련의 과정을 뜻합니다.
* **폭포수 모델 :** 소프트웨어 개발 절차 중 가장 대표적인 모델로, 각 단계가 마치 폭포가 떨어지듯 순차적으로 진행되기 때문에 붙여진 이름입니다.

**[폭포수 모델의 6단계]**
1. **프로젝트 계획 :** 무엇을 할지 계획하는 단계 
2. **업무 분석 :** 업무가 어떻게 돌아가는지 파악하고 정리하는 단계
3. **시스템 설계 :** 분석된 업무를 컴퓨터에 적용시키기 위해 알맞은 형태로 다듬는 단계
4. **프로그램 구현 :** 실제 프로그래밍 언어로 코딩하는 단계
5. **테스트 :** 프로그램에 오류가 없는지 확인하는 단계
6. **유지보수 :** 완성된 프로그램을 운영하며 보완하는 단계

## 3. 데이터베이스 모델링의 위치
* 폭포수 모델의 6단계 중에서 데이터베이스 모델링은 **2단계 '업무 분석'**과 **3단계 '시스템 설계'** 단계에 해당합니다.
* 이 단계를 거치면서 가장 중요한 데이터베이스 개체인 **테이블 구조가 최종적으로 결정**됩니다.

## 5. 데이터베이스 개체

<!-- 데이터베이스 개체에 관해 배우게 된 점을 적어주세요. -->

## 1. 데이터베이스 개체란?
* **정의 :** 데이터베이스 안에 저장되는 테이블, 인덱스, 뷰, 스토어드 프로시저 등을 통틀어 일컫는 말입니다.
* **특징 :** 데이터베이스의 가장 핵심적인 개체는 '테이블'이지만, 데이터베이스를 효율적으로 운영하고 관리하기 위해서는 테이블 외에도 다양한 개체들이 함께 사용됩니다.

## 2. 핵심 데이터베이스 개체 3가지

### ① 인덱스 (Index)
* **개념 :** '찾아보기'와 아주 비슷한 개념입니다.
* **역할 :** 데이터를 조회할 때 결과가 나오는 속도를 획기적으로 빠르게 해줍니다.
* **동작 원리 :** 인덱스가 없다면 원하는 데이터를 찾기 위해 처음부터 끝까지 전체를 뒤져야 하는 전체 테이블 검색(Full Table Scan)이 발생하여 시간이 매우 오래 걸립니다. 하지만 인덱스를 생성해 두면 빠르고 효율적으로 데이터를 찾는 **인덱스 검색**이 가능해집니다. 

### ② 뷰 (View)
* **개념 :** 한마디로 정의하면 **가상의 테이블**입니다. 윈도우 운영체제의 **바로 가기 아이콘**과 비슷한 역할을 합니다.
* **역할 :** 일반 사용자의 입장에서는 테이블과 똑같이 보이지만, 실제로는 진짜 데이터를 가지고 있지 않고 진짜 테이블에 링크(연결)만 되어 있는 형태입니다. 뷰의 실체는 사실 `SELECT` 문으로 이루어져 있습니다.
* **장점 :** 진짜 테이블에 직접 접근하는 것을 막아 **보안에 도움**이 되며, 복잡하고 긴 SQL 문을 뷰로 만들어두면 **SQL 문을 간략하게 사용**할 수 있다는 장점이 있습니다.

### ③ 스토어드 프로시저 (Stored Procedure)
* **개념 :** 여러 개의 SQL 문을 하나로 묶어주거나, SQL 안에서 일반 프로그래밍 언어(C, 자바, 파이썬 등)처럼 코딩할 수 있게 해주는 기능입니다.
* **역할 :** 매번 긴 SQL 문을 반복해서 입력할 필요 없이, 여러 줄의 SQL 문을 스토어드 프로시저로 만들어 묶어둔 뒤 필요할 때마다 호출(`CALL`)하여 한 번에 실행할 수 있습니다.

## 3. 개체의 생성과 삭제
이러한 데이터베이스 개체들을 만들고 지울 때는 공통적인 명령어를 사용합니다.
* **생성 (CREATE) :** 데이터베이스 개체를 생성할 때는 `CREATE 개체_종류 개체_이름` 형식을 사용합니다. (예: `CREATE TABLE`, `CREATE VIEW`, `CREATE PROCEDURE` 등)
* **삭제 (DROP) :** 생성한 데이터베이스 개체를 삭제할 때는 `DROP 개체_종류 개체_이름` 형식을 사용합니다. (예: `DROP TABLE`, `DROP VIEW` 등)
