## 39강. LOD

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적어주세요. -->

- **LOD 표현식이란?**
  - LOD(Level of Detail) 표현식은 Tableau에서 원하는 세부 수준(차원)에 따라 집계 계산을 할 수 있도록 해주는 기능입니다. 즉, 현재 뷰(View)에 표시된 차원과 상관없이 사용자가 지정한 차원 기준으로 계산을 수행할 수 있습니다. 

- **FIXED**
  - FIXED는 뷰에 포함된 차원과 무관하게, 사용자가 지정한 차원(예: 지역, 범주 등) 기준으로 집계를 수행합니다.
  - 예시: `{FIXED [지역]: SUM([매출])}`은 뷰에 어떤 차원이 있든 상관없이 지역별 매출 합계를 계산합니다. 
  - 차원을 입력하지 않으면 전체 데이터 기준으로 계산됩니다.

## 40. LOD EXCLUDE

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적고, 아래 두 질문에 답해보세요 :) -->

## LOD EXCLUDE의 특징
- **EXCLUDE**는 현재 뷰에서 특정 차원을 제외하여 집계를 계산할 때 사용합니다.
- 예를 들어, 'EXCLUDE [하위 범주]: SUM([매출])'은 하위 범주 차원을 무시하고 매출의 합계를 계산합니다.
- EXCLUDE를 사용하면 뷰에 포함된 다른 차원의 영향을 받아 값이 변할 수 있습니다. 즉, 뷰의 세부 수준이 바뀌면 결과도 달라집니다.
- EXCLUDE는 관련 차원을 필터로 걸었을 때 필터의 영향을 받습니다. 예를 들어, 하위 범주 필드로 필터를 걸어 특정 값을 제외하면 EXCLUDE로 계산된 값도 변경됩니다. 

## FIXED, INCLUDE와의 비교
- **FIXED**는 특정 차원을 기준으로 집계를 고정하며, 필터의 영향을 받지 않습니다.
- **INCLUDE**는 뷰에 없는 차원을 추가하여 더 세부적인 수준에서 집계를 수행합니다.

## 실습 예시
- EXCLUDE를 사용해 하위 범주 차원을 제외하고 각 제품의 매출과 액세서리 매출의 차이를 계산할 수 있습니다. 

이처럼 EXCLUDE는 뷰의 세부 수준과 필터의 영향을 받으면서 특정 차원을 제외한 집계를 할 때 유용하게 사용할 수 있습니다.

> **🧞‍♀️ FIXED와 EXCLUDE을 사용하는 경우의 차이가 무엇인가요?**

```
FIXED : 기준을 내가 강제로 고정
EXCLUDE : 현재 뷰 기준에서 일부 차원을 제거
```

> **🧞‍♀️ 왜 ATTR 함수를 사용하나요?**

```
[ 정의 ]
값을 계산하는 게 아니라 이 값이 하나인지 확인하는 함수

이유 1 : 집계 충돌 방지
이유 2 : 데이터 무결성 확인
이유 3 : 라벨에서 안전하게 사용
```



## 41. LOD INCLUDE

<!-- INCLUDE, EXCLUDE, FIXED 등 본 강의에서 알게 된 LOD 표현식에 대해 알게 된 점을 적고, 아래 두 질문에 답해보세요 :) -->

이번 강의를 통해 Tableau의 LOD(Level of Detail) 표현식에 대해 다음과 같은 점을 알게 되었습니다.

## LOD 표현식의 종류와 특징

1. **INCLUDE LOD 표현식**
   - 현재 뷰의 차원에 특정 차원을 추가하여 집계 계산을 할 수 있습니다.
   - 예를 들어, 각 도시의 고객당 평균 매출을 계산할 때, 주문 아이디 차원을 추가하여 평균을 구할 수 있습니다.
   - INCLUDE LOD는 차원 필터의 영향을 받으므로, 필터를 통해 결과값이 달라질 수 있습니다.
   - 예시: `AVG({ INCLUDE [주문 ID] : SUM([매출]) })`와 같이 사용합니다.

2. **EXCLUDE LOD 표현식**
   - 현재 뷰의 차원에서 특정 차원을 제외하고 집계 계산을 할 수 있습니다.
   - 차원 필터의 영향을 받습니다.

3. **FIXED LOD 표현식**
   - 뷰에 표시되는 차원과 상관없이 지정한 차원 기준으로 집계 계산을 할 수 있습니다.
   - 차원 필터의 영향을 받지 않습니다.
   - 예시: `AVG({ FIXED [주문 ID] : SUM([매출]) })`

## LOD 표현식 선택 기준
- 뷰에 표시되는 값이 차원이면 FIXED LOD만 사용할 수 있습니다.
- 반환값이 차원 필터에 영향을 받아야 한다면 INCLUDE 또는 EXCLUDE LOD를 사용해야 합니다.

특히 INCLUDE LOD는 뷰에 없는 차원을 추가하여 집계할 때 유용하며, 차원 필터의 영향을 받는다는 점이 중요한 특징입니다.

> **🧞‍♀️ 그렇다면 어떤 경우에 각 표현식을 사용하나요? 예시와 함께 적어보아요**


```
INCLUDE LOD - 더 세밀한 기준으로 계산할 때
✔ 사용 상황
: 현재 뷰보다 더 작은 단위로 계산한 뒤, 다시 집계할 때

EXCLUDE LOD - 일부 기준을 제거하고 비교할 때
✔ 사용 상황
현재 뷰에서 특정 차원을 제거하고, 전체 대비 비교할 때

FIXED LOD - 기준을 고정하고 싶을 때
✔ 사용 상황
뷰와 관계없이 항상 같은 기준으로 계산해야 할 때
```



## 42. 매개변수

- 매개변수는 고정된 상수 값이 아닌, 동적으로 변경 가능한 값을 제공하는 기능입니다.
- 매개변수는 반드시 계산식, 필터, 참조선 등과 함께 사용되어야 실제로 작동합니다. 단독으로는 효과가 없습니다. 
- 매개변수는 여러 가지 방법으로 생성할 수 있습니다:
  1. 필터를 통해 생성 (drop-down 메뉴에서 '새 매개변수 만들기' 선택)
  2. 원하는 필드에서 마우스 우클릭 후 '만들기 > 매개변수' 선택
  3. 데이터 패널에서 직접 생성
- 매개변수를 활용하면 사용자가 직접 값을 입력하거나 선택하여, 예를 들어 상위 N개의 제품을 동적으로 표시할 수 있습니다.
- 매개변수는 집합(Set)에도 적용할 수 있어, 집합의 기준값을 매개변수로 설정하면 값에 따라 집합의 결과가 달라집니다.

즉, 매개변수는 태블로에서 대시보드나 시각화의 유연성을 높여주는 중요한 도구입니다.

## 44. 매개변수 실습

### 매개변수란 무엇인가

> 사용자가 값을 직접 입력하거나 선택하면, 그 값에 따라 시각화가 동적으로 변경되는 변수

---

### 매개변수의 핵심 역할

**1. 사용자 입력 기반 분석 가능**

* 값(숫자, 날짜 등)을 직접 조정
* 분석 결과가 실시간으로 변경

---

**2. 계산식과 결합 가능**

EX.
IF SUM([매출]) >= [목표 매출] THEN "달성"
ELSE "미달성"
END

-> KPI 분석 가능

---

**3. 참조선과 결합**

* 매개변수 값 → 기준선 생성
* 목표값 시각화
---

**4. 날짜 범위 제어 가능**

* 시작 날짜 / 종료 날짜 매개변수 생성
* 특정 기간 강조 가능

-> 기간 분석 / 시뮬레이션 가능

---

### 매개변수 활용 사례

### 사례 1 : 목표 매출 분석

* 매개변수: 목표 매출
* 참조선: 목표선 표시
* 계산 필드: KPI 분류

---

### 사례 2: 날짜 범위 분석

* 매개변수:

  * 시작 날짜
  * 종료 날짜
* 참조 구간:

  * 선택한 기간 강조

---

### 매개변수 + 계산 필드 조합

### 핵심 구조

1. 매개변수 생성
2. 계산 필드에서 사용
3. 시각화에 반영

EX.

IF [주문 날짜] >= [시작 날짜]
AND [주문 날짜] <= [종료 날짜]
THEN [매출]
END

---

### 매개변수의 특징

| 특징     | 설명         |
| ------ | ---------- |
| 사용자 입력 | 직접 값 변경 가능 |
| 데이터 독립 | 데이터와 별개    |
| 단일 값   | 하나의 값만 가짐  |
| 동적 분석  | 실시간 변경     |

---

### 매개변수 vs 필터 차이

| 구분     | 매개변수   | 필터     |
| ------ | ------ | ------ |
| 값 개수   | 1개     | 여러 개   |
| 데이터 연결 | 없음     | 있음     |
| 역할     | 변수     | 데이터 제한 |
| 활용     | 계산/참조선 | 데이터 제거 |

## 45. 워크시트 마크카드

<!-- 마크카드에 대해 알게 된 점을 적어주세요 -->

### 개념

> 마크 카드는 뷰(View)에 표시되는 데이터의 시각적 표현(마크)을 제어하는 영역이다.

차트의 형태와 관계없이 색상, 크기, 텍스트 등의 표현을 변경할 수 있다.

### 역할
- 데이터의 시각적 표현 제어

- 동일한 데이터라도 다른 방식으로 표현 가능

- 차트의 가독성과 전달력 향상

### 주요 구성 요소
| 항목      | 역할    | 설명                     |
| ------- | ----- | ---------------------- |
| Color   | 색상    | 범주 또는 값에 따른 구분         |
| Size    | 크기    | 값의 크기 표현               |
| Label   | 텍스트   | 데이터 값 표시               |
| Detail  | 세부 정보 | 데이터 수준(Granularity) 결정 |
| Tooltip | 도구 설명 | 추가 정보 제공               |


### 특징

1) 배치 위치에 따라 의미가 달라짐

- 같은 필드라도 위치에 따라 역할이 달라진다.

>색상 → 구분

>크기 → 강조

>레이블 → 정보 표시

2. 차트 유형에 따라 옵션 변경

>라인 차트 → Path

>맵 → Halo

>원 → Shape

3) 연속형 vs 불연속형 표현 차이

>연속형 → 색상 그라데이션

>불연속형 → 범주별 색상

### 활용 예시

- 중요한 값 강조 → Color, Size
- 정보 전달 강화 → Label
- 데이터 세분화 → Detail
- 추가 정보 제공 → Tooltip

## 46. 서식 계층

<!-- 서식계층에 대해 알게 된 점을 적어주세요 -->

### 개념

Tableau에서는 시각화의 서식을 한 번에 적용하는 것이 아니라, **여러 단계(계층)를 통해 적용**됩니다.
각 계층은 서로 영향을 주며, **더 구체적인(하위) 계층이 우선적으로 적용되는 구조**를 가집니다.

---

### 계층 구조의 핵심 원리

상위 계층에서 설정한 서식이 존재하더라도,
하위 계층에서 다른 서식을 지정하면 **하위 계층의 설정이 우선 적용됩니다.**

* 전체 → 부분으로 내려갈수록 **더 강한 우선순위**를 가짐

---

### 서식 계층의 구성

* 통합 문서 (Workbook)
* 워크시트 (Worksheet)
* 행/열 (Rows / Columns)
* 특정 필드 (Field)
* 필드 레이블 (Field Label)
* 마크 (Marks)
* 제목 (Title)

---

### 적용 방식의 특징

* 통합 문서에서 글꼴을 설정하면 전체에 적용됨
* 하지만 행이나 특정 필드에서 따로 설정하면
  → 해당 부분만 별도로 표시됨

예를 들어,

* 전체 글꼴: 녹색
* 행 글꼴: 빨간색

→ 행에는 빨간색이 적용됨

---

### 필드 단위 서식

특정 필드만 별도로 서식을 적용할 수 있습니다.

방법

* 필드 우클릭 → 서식
* 또는 서식 패널에서 필드 선택

이 경우 해당 필드는 **다른 계층보다 우선 적용**됩니다.

---

### 필드 레이블 서식

필드 이름(헤더)에 대한 서식을 별도로 설정할 수 있습니다.

* 색상 변경
* 밑줄
* 글꼴 변경

데이터 값이 아닌 **제목 영역만 따로 제어 가능**

---

### 마크 서식

마크 카드에서 설정하는 서식은

* 텍스트
* 색상
* 크기

등을 직접 제어하며,

## 47. 워크시트 서식

### 개념

Tableau의 워크시트 서식은
차트의 데이터 구조를 바꾸는 것이 아니라,
**텍스트, 정렬, 색상 등을 조정하여 가독성과 전달력을 높이는 기능**이다.

---

### 주요 서식 옵션

워크시트 상단의 서식 탭에서는 다음과 같은 옵션을 제공한다.

* 글꼴 (Font)
* 맞춤 (Alignment)
* 음영 (Shading)
* 테두리 (Border)
* 라인 (Line)

---

### 글꼴 (Font)

워크시트에 표시되는 텍스트의 시각적 요소를 설정한다.

설정 가능 요소

* 글꼴 종류
* 글자 크기
* 색상

적용 대상

* 머리글(Header)
* 데이터 값(Pane)
* 총계(Total)

특징

* 머리글과 데이터 값을 **각각 다르게 설정 가능**
* 총계는 별도로 서식 적용 가능

---

### 맞춤 (Alignment)

텍스트의 정렬과 방향을 설정한다.

설정 가능 요소

* 좌/우/중앙 정렬
* 상/중/하 정렬
* 텍스트 방향

활용

* 가독성 향상
* 데이터 정렬 구조 명확화

---

### 음영 

데이터 셀의 배경 색상을 설정한다.

특징

* 행(Row), 열(Column), 셀(Cell) 단위로 설정 가능
* 교차 음영 적용 가능

---

### 구간 크기

음영의 적용 간격을 조정하는 기능

* 구간 크기 ↓ → 음영 사라짐
* 구간 크기 ↑ → 음영 간격 증가

> 음영을 적용할 “행 단위 범위”를 설정하는 기능

---

### 활용 효과

워크시트 서식을 활용하면

* 데이터 구분이 명확해짐
* 가독성 향상
* 시각적 강조 가능

---

### 핵심 정리

워크시트 서식은
데이터 자체를 변경하지 않고,
**표현 방식을 조정하여 분석 결과를 더 쉽게 전달하기 위한 기능이다.**
